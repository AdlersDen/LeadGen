1. Executive Summary
The Corporate Lead Intelligence & Outreach Automation System is an internal B2B sales tool that automates the end-to-end process of identifying corporate prospects, finding their key decision-makers, and reaching out with AI-personalized pitches.
A user inputs a geographic 
pincode
 via a React dashboard, and the system delivers a ready-to-send outreach campaign to relevant companies in that area, logging all activities directly into Google Sheets. This eliminates hours of manual research per sales cycle. The system is purpose-built for the company's B2B gifting and employee engagement vertical, targeting HR Managers, Marketing Heads, and Admin/Procurement leads at companies within a defined radius.

2. Problem Statement
Currently, the sales team identifies and reaches out to B2B prospects manually — searching Google Maps, LinkedIn, and company directories to find relevant businesses, then individually researching contacts and crafting outreach messages.
This process is:
Time-intensive:
 2–4 hours per campaign preparation.
Inconsistent:
 Pitch quality and targeting vary by individual.
Unscalable:
 Volume is limited by manual bandwidth.
Untracked:
 No centralized visibility into outreach status or results.
This system replaces the manual pipeline with an automated, AI-driven workflow that runs in minutes.

6. Feature Specifications
6.1 Module 1 — Company Discovery Engine
Converts a 
pincode
 to a list of relevant businesses, filtering out noise like restaurants and retail.
Data Sources:
 Google Geocoding API (converts 
pincode
 to coordinates), Google Maps Places API (fetches nearby businesses).
Filtering Logic:
 Include offices, companies, firms, corporate HQs. Exclude restaurants, salons, retail, hospitals, schools. Must have a Google Maps listing.
6.2 Module 2 — Contact Intelligence Engine
Finds verified decision-makers within each discovered company using third-party lookup APIs (replacing LinkedIn scraping).
Target Roles (Priority):
 HR Manager/CHRO, Marketing Head/CMO, Admin/Procurement Head.
Lookup Strategy:
 Hunter.io API (primary), Apollo.io API (fallback), Email pattern generation (last resort).
6.3 Module 3 — Data Cleaning & Normalization
Deduplication:
 Hash on company name + domain; skip duplicate records.
Standardization:
 Lowercase names, strip special characters.
Validation:
 Regex format check + MX record verification on domain.
Confidence Filtering:
 Skip contacts with Hunter.io confidence below 50.
6.4 Module 4 — AI Pitch Generation Engine
Generates a fully personalized cold outreach email using Gemini 2.5 Flash-Lite.
Rate limit:
 4-second delay between API calls to stay under 15 RPM.
Caching:
 Hash prompt inputs; skip API call if cached result exists.
Prompt Strategy:
 Instructs the model to write under 120 words, mention specific gifting/employee engagement use cases, and end with a soft CTA.
6.5 Module 5 — Email Outreach Engine
Infrastructure:
 SendGrid API with dedicated company domain (SPF/DKIM/DMARC configured).
Warm-up:
 Must send 20–50 emails/day for the first week before scaling to 300/day max.
Compliance:
 Mandatory unsubscribe link in every email footer.
6.6 Module 6 — Tracking & Storage (Google Sheets)
Metrics Tracked:
 Emails sent, delivery status, open rate, reply rate.
Storage:
 All records are appended as new rows into dedicated tabs in a central Google Sheet.
Duplicate Prevention:
 Backend script runs a 
get_all
 check against the Sheet to skip contacts emailed in the last 90 days.
